Linux est un système d'exploitation au même titre que Windows et Mac OS.
Il est réputé pour sa sécurité et ses mises à jour fréquentes.

Le principe de base de Linux est de nous permettre un contrôle total sur la machine sur laquelle il est installé.

Un système d'exploitation : superlogiciel faisant la liaison entre la partie hardware et software sur un ordinateur.
Se lance au démarrage après l'écran de boot.

Boot loader : Programme permettant de choisir un OS à lancer au démarrage. Aussi appelé "GRUB".

Les programmes Windows ne fonctionnent pas sur Linux et inversement.
Cependant il est possible de lancer des programmes windows avec le programme wine.

La majorité des programmes sous LINUX sont gratuits, souvent mis à jour et meilleurs voir exclusifs.

Le partionnement : Découper virtuellement un disque en plusieurs parties.
Réduire la mémoire allouée à un OS sur un disque dur pour en laisser de la place pour un autre.

Il est préférable d'effectuer une défragmentation avant de partionner son HDD pour y installer Linux.

Il est utile de créer une partition de 1GO max pour une extension à la mémoire vive afin que linux continue dessus en cas de dépassement de mémoire.
Définir en tant qu'espace d'échange "swap".

Il y a un système de fichier par partition. Un système de fichier est une façon d'organiser les fichiers par référence
au sein d'un annuaire, mais également de leur donner des droits. Il ne peut y avoir qu'un seul système de fichier par  partition.

Systèmes de fichiers Microsoft : FAT 16 (< à 4GO), FAT 32 (< à 2TO), NTFS (16Eo + journalisé)
Linux : ext2 (pas besoin de defragmentation car les fichiers se fragmentent peu), ext3 (journalisation), ext4 (plus performant)

Le nom des disques sous Linux :
-1ère lettre : H ou S : H pour IDE et S pour SCSI (S-ATA), ce sont deux types de connectique à la carte mère
-2ième lettre : D 
-3ième lettre : A, B, C... (premier disque, deuxième, troisième...)
-Un chiffre pour la partition : 1, 2, 3...

Linux est un OS multi-utilisateurs dédié à être utilisé par plusieurs personnes. 
Les comptes utilisateurs sont limités en droit.
Le compte administrateur root permet de tout faire et est accessible via la commande sudo (Substitute User DO). On peut se logguer en root sur beaucoup de distributions Linux mais pas
sur Ubuntu car c'est consideré comme trop dangereux pour un débutant. Cela constitue une barrière contre les virus notamment.

Lors de la création d'un compte, un groupe portant le même nom est créé.

Les droits des fichiers et dossiers sont représentés par une série de lettres : 
	d : dossier
	l : link
	r : droit de lecture
	w : droit d'écriture
	x : droit d'éxécution (programmes et scripts) (si c'est un dossier, on peut le traverser et voir son contenu descendant)
	- : fichier / pas de droit

Sous la forme suivante : 

wxxxyyyzzz : 

w : fichier / dossier / link
xxx : droits du propriétaire
yyy : droits du groupe
zzz : droits des autres utilisateurs

Chaque commandes produisent deux flux de données différentes : la sortie standard et la sortie d'erreurs.

On peut rediriger le résultat d'une commande dans un fichier au lieu de l'afficher dans la console en utilisant
les flux de redirection.

Sortie standard : > nomFichier (création / replacement) ou >> (ajout à la fin du fichier) à la fin de la commande.
Sortie d'erreurs : 2> nomFichier (création / remplacement) ou 2>> nomFichier (ajout à la fin du fichier) ...
Fusionner les deux sorties dans un fichier : >(>) nomFichier 2>&1

On peut également agir sur les entrées d'une commande :

< : lire depuis un fichier (retourne son contenu)
<< FIN : lire depuis le clavier (on peut remplacer FIN par ce que l'on veut, c'est ce qu'il faut taper pour sortir)

| permet de chainer des commandes.

On peut écrire plusieurs commandes sur une seule ligne en les séparant par un ; ou un && (celui-ci garantit la bonne execution).

===================== Connexion à un serveur distant avec SSH ====================

Un serveur est un PC allumé et connecté à internet 7j/7 24h/24.

Sur un réseau, les ordinateurs communiquent via un protocole. Il en existe plusieurs en fonction de ce que l'on souhaite se transmettre.
Ex : IMAP (mail), FTP (fichiers), HTTP (pages web)...
SSH est un protocole de communication chiffré. Successeur de Telnet qui envoyait les messages en clair sur le réseau.
Il utilise une méthode de chiffrement symétrique et asymétrique.

Grâce à SSH, on peut se connecter à un serveur linux distant de manière sécurisée.

Transformer son pc en serveur : apt-get install openssh-server
A partir de là, le serveur SSH sera lancé à chaque démarrage. Sinon, utiliser la commande :

sudo /etc/init.d/ssh start -> démarrer ssh
sudo /etc/init.d/ssh stop -> arrêt de ssh

Pour se connecter à un serveur, il faut utiliser la commande ssh :

ssh login@ip
Le mot de passe sera demandé après un message indiquant l'empreinte du serveur (si connecté pour la première fois).

Sur un PC (windows), utiliser le logiciel PuTTY pour se connecter sur un serveur linux.

===================== SCRIPT SHELL =====================


Minilangage de programmation dédié à Linux pour automatiser certaines tâches.

Shell = environnement console. Programme qui gère l'invite de commande.
SH (le plus ancien présent sur tout UNIX) et Bash (présents sur Linux et MAC OS X, qui hérite de SH) sont les shells les plus répendus.

.bashrc est le fichier de configuration du shell bash.

Les script diffèrent d'un shell à l'autre.

Par convention, les fichiers de script shell portent l'extension .sh mais ce n'est pas obligatoire.

SCRIPT BASH (plus complet que SH et plus répendu que ses alternatives plus puissantes) :

Indiquer en première ligne du fichier le chemin du shell dont on veut forcer l'utilisation : #!/bin/bash
Si cette ligne n'est pas présente, le shell que l'utilisateur utilise sera utilisé.

#! = sha-bang

Ecrire ensuite les commandes à automatiser.

Une fois le fichier enregistré, il faut lui donner le droit d'éxecution.

Façon rapide de faire : chmod +x nomFichier.sh

Executer le script : ./nomFichier.sh

Débugguer un script : bash -x nomFicher.sh

Pour lancer le script de partout, il faut le placer dans un repértoire du PATH comme /bin

Déclaration de variable :

nomVariable='valeur'

/!\ Ne pas mettre d'espace avant et après le =

Dans BASH, tout est chaine de caractère.

Afficher une variable : 

echo $nomVariable

Différents types de quotes (délimiteurs) :

'' -> chaine de caractère pure
"" -> analyse ce qu'il y a dedans (affiche texte et contenu des variables)
`` -> execute ce qu'il y a dedans (une nouvelle méthode est apparue : $() )

Lire la saisie utilisateur :

read variable
read -p 'texte' variable
read -p 'Entrez un mdp' -s password
read -p 'entrez un message avant 5 secondes :' -t 5 message
read -p 'entrez votre nom (5 caracteres max) :' -n 5 nom

Faire des calculs :

let "a = 5"
let "b = 2"
let "c = a + b"

Variables globales (d'environnement) sont disponibles partout :

les afficher avec la commande env
leur nom sont en majuscule par convention
on peut les utiliser dans notre code ou en créer de nouvelles avec export

Créer une variable tableau :

tableau=('valeur1' 'valeur2' 'valeur3')

Accéder à une case du tableau :

${tableau[indice]}
echo ${tableau[*]} // affiche tous les éléments du tableau séparés par un espace

On peut passer des paramètres à la commande du script, ils sont récupérés dedans à travers des variables partant de $1 à $9. ($0 contient le nom du script.)
Utiliser shift pour décaler à droite (si + de 9 paramètres).

Conditions :

if [ test ]
then
	instructions...
elif [ test ]
then
	instructions...
else
	instructions...
fi

OU

if [ test ]; then
	instructions...
elif [ test ]; then
	instructions...
else
	instructions...
fi

Dans le test, -z $chaine vérifie que la chaine est vide (ou n'existe pas), -n $chaine vérifie qu'elle est non vide (ou existe).

/!\ Il y a un espace après le [ et un espace avant le ] du test.

Pour comparer des nombres, il faut utiliser les paramètres -eq (égalité), -ne (différent), -lt (inférieur), -gt (supérieur), -le (inférieur ou égal), 
-ge (supérieur ou égal)

Pour faire des tests sur un fichier, il faut utiliser les paramètres -e $fichier (fichier existe), -d $fichier (vérifie si dossier), 
-f $fichier (vérifie si fichier), -L $fichier (lien symbolique), -r $fichier (si lisible), -w $fichier (si modifiable), -x $fichier (si executable), 
$fichier -nt $fichier2 (si plus récent que), $fichier -ot $fichier2 (si plus ancien que).
