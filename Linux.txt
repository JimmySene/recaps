Linux est un système d'exploitation au même titre que Windows et Mac OS.
Il est réputé pour sa sécurité et ses mises à jour fréquentes.

Le principe de base de Linux est de nous permettre un contrôle total sur la machine sur laquelle il est installé.

Un système d'exploitation : superlogiciel faisant la liaison entre la partie hardware et software sur un ordinateur.
Se lance au démarrage après l'écran de boot.

Boot loader : Programme permettant de choisir un OS à lancer au démarrage. Aussi appelé "GRUB".

Les programmes Windows ne fonctionnent pas sur Linux et inversement.
Cependant il est possible de lancer des programmes windows avec le programme wine.

La majorité des programmes sous LINUX sont gratuits, souvent mis à jour et meilleurs voir exclusifs.

===================== HISTOIRE =====================

1969 -> UNIX plus puissant que MS-DOS mais plus compliqué
1984 -> MS-DOS de Microsoft

Texte blanc sur fond noir.

C'est en 1984 que Richard Stallman fonda le projet GNU : un système d'exploitation basé sur UNIX gratuit et libre.
(MAC OS est également basé sur UNIX.)

PS : Le code source d'Unix n'a pas été repris et a complètement été réécrit. Il n'y a que le fonctionnement qui est le même.

1991 -> Linus Torvalds créé son propre OS : LINUX.

Le projet GNU et Linux ont fusionné pour créer GNU/Linux.

===============================================================

Différentes distributions de Linux ont été créés. Ce qui différe d'une distribution à l'autre est :

-L'installation de l'OS
-La gestion de l'installation des programmes
-Les programmes préinstallés

Debian et Red Hat sont des distributions de Linux et il en existe d'autres.

L'un des gros avantages de Debian est son gestionnaire de paquets apt-get. 
Tous les programmes téléchargeables sont centralisés au même endroit et sont facilement installables et desinstallables.
Debian est l'une des seules distrib à être développée par des indépendants à travers le monde et non par une entreprise.

Il existe également des distributions de distributions. 
Skolelinux (pour les écoles) et Ubuntu sont des distributions de Debian, il en existe d'autres.

Ubuntu est devenu célèbre pour plusieurs raisons :
-Prévu pour le grand public, d'où son slogan "Linux for human beings".
-Mises à jour fréquentes
-Beaucoup d'utilisateurs donc beaucoup d'aide

Le programme X gère le mode graphique de Linux. Par dessus y est greffé un gestionnaire de bureau.
Celui-ci gère les fenêtres, leur apparence, leurs options...

Les plus connus sont Unity, KDE, XFCE et Gnome.
Il existe différentes versions d'Ubuntu liés à un gestionnaire de bureau différent comme Kubuntu (KDE) ou Xubuntu (XFCE).
Ubuntu était livré par défaut avec Gnome, puis avec Unity pour enfin revenir sur Gnome.

Il est possible d'avoir plusieurs gestionnaires de bureaux et d'en faire le choix au démarrage après s'être logué.
La compatibilité des programmes reste optimale.

La console reste la même peu importe le gestionnaire de bureau choisi.

(Sur Unity : On peut changer de gestionnaire de bureau en cliquant sur l'icone Ubuntu à côté de login / mot de passe.)
(Sur KDE : La petite flèche pointant vers le bas sur l'écran de login)

Le partionnement : Découper virtuellement un disque en plusieurs parties.
Réduire la mémoire allouée à un OS sur un disque dur pour en laisser de la place pour un autre.

Il est préférable d'effectuer une défragmentation avant de partionner son HDD pour y installer Linux.

Il est utile de créer une partition de 1GO max pour une extension à la mémoire vive afin que linux continue dessus en cas de dépassement de mémoire.
Définir en tant qu'espace d'échange "swap".

Il y a un système de fichier par partition. Un système de fichier est une façon d'organiser les fichiers par référence
au sein d'un annuaire, mais également de leur donner des droits. Il ne peut y avoir qu'un seul système de fichier par  partition.

Systèmes de fichiers Microsoft : FAT 16 (< à 4GO), FAT 32 (< à 2TO), NTFS (16Eo + journalisé)
Linux : ext2 (pas besoin de defragmentation car les fichiers se fragmentent peu), ext3 (journalisation), ext4 (plus performant)

Le nom des disques sous Linux :
-1ère lettre : H ou S : H pour IDE et S pour SCSI (S-ATA), ce sont deux types de connectique à la carte mère
-2ième lettre : D 
-3ième lettre : A, B, C... (premier disque, deuxième, troisième...)
-Un chiffre pour la partition : 1, 2, 3...

L'utilisation de la console permet d'aller beaucoup plus vites dans ce que l'on fait.

Aller dans la console : CTRL + ALT + FX (1 à 6)
Sortir de la console : CTRL + ALT + F7

Il est cependant plus pratique d'utiliser un terminal en mode graphique.

Utiliser Putty pour se connecter à un serveur Linux distant via Internet avec le protocole SSH (plus sécurisé que TelNet).
Un serveur est un ordinateur constamment connectée à Internet servant généralement à stocker et livrer des fichiers.

L'invite de commande s'affiche et indique :
nomUtilisateur@nomMachine:~$
~ = dossier "home" (dossier personnel)
$ = connecté en utilisateur "normal". # = connecté en superutilisateur "root".

Certaines commandes ont des paramètres. Il existe deux types de paramètres :
-courts : un tiret + une lettre
-longs : deux tirets + plusieurs lettres

On peut attribuer une valeur à un paramètre :
-court : parametre valeur
-long : parametre=valeur

Les commandes et paramètres sont sensibles à la casse.
Toutes les commandes sont des programmes utilisables à partir de l'invite de commande.

Autocomplétion : 
-appuie deux fois sur tab pour afficher une liste des commandes 
-appuie une fois sur tab pour autocomplétion

Si la liste s'étale sur plusieurs pages : espace pour passer à la page suivante, entrée pour la ligne et q pour sortir.

Appuie sur la touche haut pour revenir à la commande précédente, bas pour la commande suivante.

La convention de nommage pour les fichiers et dossiers : minuscules sans espaces ni accents (ce n'est pas une règle)

Dans Linux, tout est fichier. Les lecteurs C:, D:, E:... de Windows sont tous regroupés en un seul et même endroit appelé
"racine" : /. 

Se rendre dans le dossier personnel : cd:~ ou cd
Se rendre dans le dossier racine : cd /

Les dossiers personnels de chaque utilisateur se situent dans /home

Les fichiers sur Linux sont stockés en deux parties : le nom et le contenu (inode).

Linux est un OS multi-utilisateurs dédié à être utilisé par plusieurs personnes. 
Les comptes utilisateurs sont limités en droit.
Le compte administrateur root permet de tout faire et est accessible via la commande sudo (Substitute User DO). On peut se logguer en root sur beaucoup de distributions Linux mais pas
sur Ubuntu car c'est consideré comme trop dangereux pour un débutant. Cela constitue une barrière contre les virus notamment.

Lors de la création d'un compte, un groupe portant le même nom est créé.

Les droits des fichiers et dossiers sont représentés par une série de lettres : 
	d : dossier
	l : link
	r : droit de lecture
	w : droit d'écriture
	x : droit d'éxécution (programmes et scripts) (si c'est un dossier, on peut le traverser et voir son contenu descendant)
	- : fichier / pas de droit

Sous la forme suivante : 

wxxxyyyzzz : 

w : fichier / dossier / link
xxx : droits du propriétaire
yyy : droits du groupe
zzz : droits des autres utilisateurs

On peut modifier quelques paramètres des consoles en éditant le fichier bashrc situé dans etc/ ou home/.
le fichier profile fait la même chose mais pour les shell demandant de se logguer. profile fait appel à bashrc.
Modifier bashrc est donc suffisant.
Le fichier .bashrc est executé au démarrage de la console.

---

Sous les distributions Debian, il n'y a pas de programmes d'installation .exe comme sur Windows, mais des paquets en .deb.
Une sorte de dossier zippé contenant tous les fichiers du programme. 

Tous les .deb sont situés à un seul et même endroit appelé dépôt ou repository et possèdent une gestion de dépendance.
Il existe plusieurs dépôts qui sont des copies des autres proposant les même paquets.
Il est conseillé de choisir un dépôt se situant à proximité.

Le dépot Ubuntu pour la france : fr.archive.ubuntu.com

===================== COMMANDES =====================

date : affiche la date et l'heure
	MMJJHHmmAAAAA : change la date
ls : affiche la liste des dossiers et fichiers
	-a / --all : affiche les fichiers cachés (commencant par un .)
	-l : liste détaillée
	-h : affiche le poids 
	-t : trie par date de dernière modification
history : affiche un historique des commandes utilisées
clear (ctrl + l) : nettoie la console
exit (ctrl + d) : ferme la console / sort du mode root
pwd : affiche le dossier actuel
which nomCommande : connaitre l'emplacement de la commande
cd : se déplacer dans l'arborescence (chemins relatifs ou absolus)
du : affiche la taille des dossiers dans le dossier actuel
	-a : et des fichiers
	-h : poids en MO
	-s : affiche le poids total
cat nomFichier : affiche le contenu d'un fichier
	-n : affiche les numéros de ligne
less nomFichier : affiche le contenu d'un fichier page par page
head nomFichier : affiche les premières lignes d'un fichier
	-n nbLignes : affiche les X premières lignes
tail nomFichier : affiche les dernières lignes d'un fichier (CTRL + C pour fermer la commande)
	-n : Affiche les X dernière lignes
	-f : Suit la fin du fichier (temps réel)
touch nomFichier : créer un fichier (ou édite sa date de modification)
mkdir nomDossier : créer un dossier
	-p arborescence1/arborescence2/... : créer des dossiers enfants
cp nomFichier nomNouveauFichierOuCheminRelatifOuAbsolu : copie un fichier
	-R : copie un dossier et son contenu
mv nomFichierOuDossier/ cheminRelatifOuAbsolu : déplacer un fichier ou le renomme
rm nomFichier : supprimer un fichier
	-i : demande une confirmation
	-f : détruit à coup sûr
	-v : fait parler la commande
	-r : supprimer un dossier
ln nomFichierSource nomFichierCible : créer un lien physique vers un même inode
	-s : créer un lien symbolique vers un nom de fichier
sudo : passer temporairement en root
	su : en permanence
	-E : Préserver les variables d'environnement
chmod nbDroit nomFichier : modifier les droits utilisateur d'un fichier
	Manière absolu : r = 4 ; w = 2 ; x = 1 -> il faut additionner les droits et mettre 3 chiffres pour utilisateur, groupe et autres
	Manière relative : u = user, g = group, o = other ; + = ajouter, - = retirer, = = affecter -> Exemple : chmod g+w rapport.txt
	(777 = tous les droits pour tous, 000 = aucun droits pour tous)
	root outrepasse tous les droits
	-R : affecter récursivement à tous les sous-dossiers
setxkbmap fr : change le clavier en AZERTY
nano : ouvre l'éditeur de texte nano (^ signifie CTRL)
	CTRL + O pour enregistrer
	-m : permet l'utilisation de la souris dans la console (mouse)
	-i : indentation automatique (autoindent)
	-A : retour intelligent avec le bouton origine (smarthome)
	Le fichier de configuration personnel de Nano s'appelle .nanorc et est situé dans le dossier personnel
	Le fichier de configuration global de Nano s'appelle nanorc et est situé dans le dossier etc (modifiable en root)
alias : affiche la liste des alias de commandes créés dans le fichier .bashrc
apt-get : télécharger et installer des paquets
	update : mettre à jour le cache (root)
	install nomPaquet : télécharge et installe un paquet
	remove nomPaquet : supprimer un paquet
	autoremove nomPaquet : supprimer un paquet et ses dépendances devenues inutiles
	upgrade : mettre à jour ses paquets (selon ce qui est situé dans le cache, faire un apt-get update avant de préférence) 
	Modifier les dépots en éditant le fichier sources.list situé dans /etc/apt/
apt-cache search nomPaquet : rechercher un paquet si on ne connait pas son nom
man nomCommande : affiche le manuel d'une commande (man pour "manual") 
	Utiliser le / pour faire des recherches dans le manuel
	Les mots écrits en gras sont à recopier, ceux soulignés sont à remplacer
	... signifie que l'entrée est à répéter autant de fois que l'on veut
	[] signifient des arguments facultatifs contrairement aux arguments obligatoires écrits en brut
	| signifie OU
apropos motClef : retrouver une commande grâce à un mot clé (affiche une liste)
whatis nomCommande : affiche la description d'une commande
lsb_release : affiche la version actuelle de Debian
	-a : affiche les infos
	-ds : affiche une description
dpkg : Package manager pour Debian
	-L name Paquet : affiche la liste des fichiers d'un paquet
dpkg-reconfigure keyboard-configuration : Modifier le clavier
ip addr : affiche l'adresse IP
locate nomFichier : Localise un fichier dans la base de données de fichiers du disque (mise à jour une fois par jour)
find où quoi queFaireAvec : Trouver un fichier dans le disque dur à partir du dossier actuel (si pas de "où")
	-name "nomFichier" : rechercher un fichier par nom
	-size +10M (k, M, G...) (+ ou -) : recherche un fichier en fonction de son poids
	-atime X (remplacer X par le nb de jours) : recherche un fichier par date de dernier accès
	-type X (remplacer X par d si dossier, f si fichier)

	-printf "" : afficher un texte formaté
	-delete : supprime les fichiers
	-exec commande {} \; : execute une commande ({} est remplacé par le nom du fichier)
hostname : affiche le nom du pc (pour le modifier il faut modifier les fichiers en root situés dans /etc/hostname et /etc/hosts)
grep texte nomFichier : recherche un mot dans un fichier
	-i : ne tient pas compte de la casse
	-n : connaitre le numéro de ligne
	-v : inverser la recherche (ne contient pas le mot)
	-r (rgrep) : rechercher dans tous les fichiers et sous-dossiers (mettre repertoire à la fin à la place du nomFichier)
	-E (egrep) : utiliser une expression régulière comme filtre
sort nomFichier : trie le contneu d'un fichier
	-o : enregistrer le trie : sort nomNewFichier nomFichier
	-r : trier en ordre inverse
	-R : trier aléatoirement
	-n : trier des nombres
wc nomFichier : compte le nombre de lignes, mots et octets d'un fichier
	-l : compte le nb de lignes
	-w : compte le nb de mots
	-c : compte le nb d'octets
	-m : compte le nb de caractères
uniq nomFichier : supprime les doublons (nécessite que le fichier soit trié avec la commande sort)
	-c : affiche le nb de fois qu'une occurence apparait
	-d : afficher uniquement les lignes en double
cut -c nbCaracteres nomFichier : couper chaque lignes selon le nombre de caractères
	-d : indiquer un délimiteur
	-f : indiquer le numéro du ou des champs à couper

passwd nomUser : Changer le mot de passe d'un compte utilisateur

	-g : changer le groupe de l'utilisateur
	-G : changer les groupes de l'utilisateur (séparés par des virgules)
	-a : ajoute l'utilisateur à 1 ou des gorupe(s)
	EXEMPLE : usermod -G amis, prof patrick

chown nomUser nomFichier : changer le propriétaire d'un fichier
	-R : affecter récursivement les sous-dossiers et fichiers
	ASTUCE : chown permet de changer le groupe propriétaire, il suffit d'ajouter à côté du nomUser:nomGroupe
chgrp nomGroupe nomFichier : changer le groupe propriétaire d'un fichier
updatedb : met à jour la base de données de fichiers
