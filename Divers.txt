Multiplexage :
	-Faire passer plusieurs informations à travers un seul support de transmission.
Shadow DOM : 
	-Permet d'encapsuler du JavaScript et du CSS au sein d'un Web Component
	-Séparation du DOM principal
	-Utilisable seul en dehors du component
	Utiliser la méthode .attachShadow({mode: "open"}) sur un élément pour appliquer le SHADOW DOM dessus.
	Accessible via le noeud racine ShadowRoot.




Serveur : Ordinateur relié à Internet ayant pour rôle de servir des données à celui qui lui en demande.




Sérialisation : Consiste à transformer une structure de donnée simple ou complexe en fichier texte.

Formats de sérialisation : 

-Format CSV :
	"Comma Separated Value". Les données sont séparées par des virgules.
	Fichiers principalement générés par Excel et Calc (Tableurs).
-Format YAML : (Yet Another Markup Language)
	standard de sérialisation de données pour tous les langages, facile à comprendre pour les humains
	Permet d'inclure plusieurs documents en les séparant par ---
	S'utilise davantage pour la configuration de fichiers.
	Continet des types avancés comme les dates
	Commentaires : #
	Exemple :
	---
	nom_section:
		nom: valeur
		nom: valeur

	autre_section:
		nom: valeur
		...
-Format XML : 
	reine des formats de sérialisation. Pro mais verbeux, fragile et complexe.
	Données entre balises <>.
-Format JSON (Javascript Object Notation) :
	Représentation des données sous forme de variables javascript. Plus légère que XML. Pas de commentaires.
	Subset de YAML.
	S'utilise davantage pour l'envoi de données.
	Se présente sous cette forme :
	[
		{
			'nom' : 'valeur',
			'nom' : 'valeur'
		},
		{
			...
		}
	]
-Format INI
	Souvent des fichiers de configuration avec des paramètres
	Constitué de deux éléments : l'en-tête (ou section) s'écrivant [nom_section] ; et la valeur s'écrivant nom=valeur
	Exemple :
	[nom_section]
	nom=valeur
	nom=valeur

	[autre_section]
	nom=valeur
	...
