Multiplexage :
	-Faire passer plusieurs informations à travers un seul support de transmission.
Shadow DOM : 
	-Permet d'encapsuler du JavaScript et du CSS au sein d'un Web Component
	-Séparation du DOM principal
	-Utilisable seul en dehors du component
	Utiliser la méthode .attachShadow({mode: "open"}) sur un élément pour appliquer le SHADOW DOM dessus.
	Accessible via le noeud racine ShadowRoot.

Web : Ensemble de documents et services disponibles sur Internet accessibles via le protocole HTTP(s).

Client web : navigateur (Firefox, Edge, Chrome...). Envoie et recoit des requêtes (demandes) d'un serveur web.

Serveur : Ordinateur connecté en permanence à Internet, ayant pour rôle de servir des données à celui qui lui en demande.

Serveur web : serveur qui sert les services et données du web. Souvent composé d'un serveur http, d'un interpréteur de script et d'un SGBD.

Serveur http (ex : Apache) : logiciel qui permet à un serveur web de communiquer avec un navigateur en utilisant le protocole HTTP(s).

HTTP : protocole d'échange de données du web utilisant le port 80

HTTPS : protocole d'échange de données chyffrées du web utilisant le port 443

API (Application Programming Interface) : Ensemble normalisé de classes, méthodes et fonctions qui sert de façade par laquelle
un logiciel offre des services à un autre programme. On l'appelle aussi "programme fournisseur" pouvant être utilisé par le "programme consommateur".

Sérialisation : Consiste à transformer une structure de donnée simple ou complexe en fichier texte.

Formats de sérialisation : 

-Format CSV :
	"Comma Separated Value". Les données sont séparées par des virgules.
	Fichiers principalement générés par Excel et Calc (Tableurs).
-Format YAML : (Yet Another Markup Language)
	standard de sérialisation de données pour tous les langages, facile à comprendre pour les humains
	Permet d'inclure plusieurs documents en les séparant par ---
	S'utilise davantage pour la configuration de fichiers.
	Continet des types avancés comme les dates
	Commentaires : #
	Exemple :
	---
	nom_section:
		nom: valeur
		nom: valeur

	autre_section:
		nom: valeur
		...
-Format XML : 
	reine des formats de sérialisation. Pro mais verbeux, fragile et complexe.
	Données entre balises <>.
-Format JSON (Javascript Object Notation) :
	Représentation des données sous forme de variables javascript. Plus légère que XML. Pas de commentaires.
	Subset de YAML.
	S'utilise davantage pour l'envoi de données.
	Se présente sous cette forme :
	[
		{
			'nom' : 'valeur',
			'nom' : 'valeur'
		},
		{
			...
		}
	]
-Format INI
	Souvent des fichiers de configuration avec des paramètres
	Constitué de deux éléments : l'en-tête (ou section) s'écrivant [nom_section] ; et la valeur s'écrivant nom=valeur
	Exemple :
	[nom_section]
	nom=valeur
	nom=valeur

	[autre_section]
	nom=valeur
	...

Dart (anciennement appelé "Dash") : Langage de programmation créé par Google en 2011, visant à remplacer Javascript. 
Cependant, le but initial semble avoir été abandonné et la priorité est que le code dart soit compilé en JS sur tous les navigateurs.

Typescript : Langage de programmation créé par Microsoft en 2012, visant à améliorer et sécuriser le développement en langage Javascript.
Il permet le typage statique, la création de classes et d'interfaces et l'implantation de modules en gardant l'approche non contraignante du JS.

EcmaScript : Standard du web et ensemble de normes imposées aux langages de script. Langage de programmation orienté prototype.

Programmation orienté prototype : Programmation orienté objet sans classes. Un prototype est un objet à partir duquel on peut créer d'autres objets.

Langage de haut niveau : Langage avec un haut niveau d'abstraction et compréhensible pour un humain.

Langage de bas niveau : Langage avec un bas niveau d'abstraction et difficilement compréhensible pour un humain, qui se rapproche plus du langage machine.

Assembleur : Langage de bas niveau représentant le langage machine lisible pour un humain.

Compilateur : Programme qui transforme un code source en un langage de bas niveau pouvant être lu par la machine.

Décompilateur : Programme qui transforme du langage de bas niveau en langage de haut niveau.

Transcompilateur (compilateur source à source) : Programme qui traduit un code source vers un autre langage de programmation (haut niveau).

Origine : Combinaison d'un protocole, un hôte et un numéro de port

Convention de nommage : règles concernant l'écriture du code source notamment le nom des variables
	-Le Camel case : les mots sont liés sans espace avec une majuscule en début de mot, à l'exception du premier. C'est la convention la plus utilisée. [Javascript, Java, C++, C#,...]
	-Le Pascal case : les mots sont liés sans espace avec une majuscule en début de mot [Pascal, class PHP]
	-Le Kebab case / Spinal case : les mots sont en minuscules et liés par des tirets - (à utiliser dans les url, nom de fichiers, class et id html)
	-L'Underscore case / Snake case : les mots sont en minuscules et liés par des underscores _ [PHP, Ruby, Python]
	-Screaming snake case : les mots sont en majuscules et liés par des underscores _ [Constantes PHP, exceptions ADA]

Type MIME : Internet Media Type, aussi appelé Content-type est un identifiant de format de données sur internet en deux parties séparées par un /.
Les deux parties sont le type et le sous-type.
Exemples : image/jpeg, text/html, video/mp4, audio/mpeg, multipart/alternative, application/javascript...
	
CDN (Content Delivery Network) : Ensemble de serveurs situés partout dans le monde qui délivre le même contenu avec un système de cache optimal.

IHM (Interface Homme-Machine) : Moyen utilisé pour faire communiquer l'homme avec la machine.

CLI (Command Line Interface) : IHM en mode texte, basé sur des lignes de commandes.

GUI (Graphical User Interface) : IHM en mode graphique, en présence de pictogrammes et d'un dispositif de pointage.

RPM (Redhat Package Manager) : Gestionnaire de paquets (.rpm) pour les distributions LINUX Redhat, CentOS et Fedora (écrit en C).

YUM (Yellowdog Updater Modifier) : Surcouche de RPM gérant les téléchargements des dépendances (écrit en Python 2).

DNF (Dandified YUM) : Evolution de YUM ajouté dès la verison 18 de Fedora (écrit en Python 3).

APT (Advanced Package Manager) : Gestionnaire de paquets (.deb) pour les distributions LINUX Debian et ses dérivés (écrit en C++).

NPM : Gestionnaire de paquets pour Node.

Alpine (Linux) : Distribution très légère de Linux

Docker : Plateforme qui permet la virtualisation d'environnements de développements (appelés containers)

Swape space (espace d'échange) : Mémoire virtuelle située dans le disque dur servant à la décharge de RAM lorsque celle-ci arrive à saturation.

Mémoire virtuelle : RAM + Espace d'échange

Hibernation : Conserve l'ordinateur en état lors de la mise hors tension de celui-ci grâce à l'espace d'échange (Nécessite autant d'espace que de RAM).

Serveur DNS (Domain Name System) : Service permettant de faire la relation entre un nom de domaine et une adresse IP.
En entrant un nom de domaine sur un navigateur, une requête DNS est envoyée au serveur DNS (souvent situé sur la box internet)
 pour récupérer l'adresse IP du serveur web, afin de requêter celui-ci pour récupérer la page.

Serveur DHCP (Dynamic Host Configuration Protocol) : Service délivrant des adresses IP aux ordinateurs voulant se connecter à un réseau.
Permet donc d'attribuer des adresses IP dynamiques. Contrairement aux IP fixes qui sont attribuées à un ordinateur en particulier, les
adresses dynamiques sont attribuées à n'importe quel appareil à condition qu'il y ait encore de la place sur le réseau. Une plage
d'adresses appelées "l'étendue", est configurée à l'avance. Le serveur DHCP renvoie aussi d'autres informations comme l'adresse du serveur
DNS.

Adresse IP (Internet Protocol) : Numéro unique permettant à un appareil de communiquer sur un réseau et d'être identifié.
Une adresse IP est liée à la carte réseau de l'appareil connecté.

Adresse de loopback (boucle locale) : 127.0.0.1 (localhost) redirigant vers la machine elle même sur le réseau.
