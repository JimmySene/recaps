-Une API (Application Programming Interface) est un service mis à disposition dont le fonctionnement est "caché", livrée avec une interface utilisable et normée.
-API Web (Webservice) est une API utilisant le protocole HTTP.

-REST (Representational State Transfer) est un style d'architecture (ensemble de bonnes pratiques) d'écriture de reqêtes pour accéder à des webservices.
A l'instar de SOAP (Simple Object Access Protocol) qui est une alternative plus ancienne et plus complexe utilisant le XML. REST a été créé pour simplifier la démarche de communication avec le serveur en utilisant l'URL.
	- Utiliser les verbes (méthodes) existants : GET - POST - PUT - UPDATE...
	- Lien sous la forme /../../..
	- Body possibilité de mettre en JSON
	- Format de sortie XML, CSV (Command Separated Value), RSS (Really Simple Syndication) ou JSON (Javascript object notation).

Une API REST est une API qui respecte les normes du W3C. On dit que l'API est RESTful.
Validée par plusieurs personnes et non pas par une seule personne, groupe ou entreprise.

Construire une requête :

VERBE HTTP lien HTTP/X.X

Entête (date, referer, user-agent...)

-Le HTTPS crypte les données envoyées du client au serveur et vice-versa.
Il n'y a que le client et le serveur dès réception des fichiers qui peuvent voir le code source.
Attention cependant, le https ne chiffre pas les données envoyées à la BDD car les données sont déjà déchiffrées par le serv.
